## 1. QA 자동화 아키텍처 (Automation Architecture)
본 프로젝트는 **'코드 변경 → 자동 테스트 → 결과 분석 → 배포 승인'**의 유기적인 품질 관리 체계를 지향합니다.

### 1-1. 테스트 피라미드 계층 (Testing Layers)
- **단위 테스트 (Unit Test)**: 개별 함수 및 로직의 독립성 검증 (`pytest`, `Jest`)
- **통합 테스트 (Integration Test)**: 모듈 간 연결 및 API 서버 응답성 검증 (`httpx`, `Supertest`)
- **E2E 테스트 (End-to-End)**: 실제 브라우저 환경에서의 전체 사용자 여정 재현 및 검증 (`Playwright`)

### 1-2. CI 파이프라인 연동 전략
- **트리거**: Push 및 Pull Request(PR) 생성 시 자동 실행
- **검증**: 코드 커버리지(Coverage) 측정 및 테스트 결과 수집
- **결과 분기**: 
    - **실패 시**: 실시간 알림 전송 및 원인 분석 (Jira 티켓 연동 시뮬레이션)
    - **성공 시**: 품질 승인(Sign-off) 리포트 발행 및 다음 단계(배포 준비) 이행

## 2. 자동화 중심 QA 업무 흐름 (SDET STLC Workflow)
최신 자동화 트렌드에 맞춰 파이프라인 기반의 8단계 소프트웨어 테스트 생명 주기를 준수합니다.

1. **요구사항 분석**: 기획서 검토 및 자동화 가능 범위(UI/API) 확정
2. **테스트 계획 수립**: 파이프라인 연동 전략(CI/CD) 및 도구 선정
3. **자동화 스크립트 코딩**: 수동 TC를 기반으로 `Playwright` / `Pytest` 자동화 테스트 코드 작성
4. **동적 테스트 환경 구축**: `docker-compose`를 활용하여 실제와 동일한 샌드박스 환경 세팅 및 더미 데이터 적재
5. **파이프라인 실행**: CI/CD 연동을 통해 자동화 스크립트 전수 검사 실행 및 탐색적 테스트 병행
6. **자동 결함 리포팅**: 에러 발생 시 `Trace Viewer` 기록과 함께 로그 자동 첨부 및 이슈 트래킹
7. **무한 회귀 테스트 (Regression)**: 코드 수정 시 전체 자동화 파이프라인 재가동으로 사이드 이펙트 완벽 차단
8. **자동 배포 승인 (Sign-off)**: 파이프라인 Pass율 100% 달성 시 품질 보고서 발행 및 운영 서버 배포 확정

## 3. 상세 테스트 범위 (Scope)

### 3-1. UI / E2E 테스트 (핵심 관문)
- **대시보드**: 히트맵 시각화 정합성 및 카드 노출 상태
- **뉴스 아카이브**: 검색, 필터링 및 대량 데이터 로딩 반응성
- **AI 챗봇**: RAG 엔진 답변 품질 및 출처 링크 무결성

### 3-2. 시스템 회복력 (Resilience)
- **API 장애 대응**: 백엔드 500 에러 시 UI 사용자 경험 유지
- **네트워크 지연**: 지연 로딩 시 스켈레톤/로딩바 노출 여부

---

## 4. 환경 및 결함 관리 (Environment & Defect)
- **도구**: Playwright (UI), Pytest (API)
- **리포팅**: HTML Reporter, Trace Viewer (Trace.zip), Video Logs
- **승인 기준**: 주요 기능(P0~P1) Pass율 100% 달성 시 Sign-off
